# DTL P1 — 핵심 운영 원장 (구현 정합본)

- 기준일시: 2026-05-28
- 범위: 홈, 업무수행현황, 프로젝트 관리, 프로젝트 상세, 진행이력
- 기준: 현재 코드 구현 (`apps/web`, `apps/api`) 우선

---

## 1. 홈 (`/`)

### 1.1 화면 구성
- 핵심 현황 카드
  - 1행: 현재 인원, 수행 인원, 제안 인원, 대기 인원, 가동률, 가득률
  - 2행: 신규 프로젝트, 완료 예정 프로젝트, 완료 프로젝트, 인력 변동, WIN, LOSS
- 기간별 가동률 / 가득률 차트
- 인력 현황(본부 합계 행 포함)
- 가동률 / 가득률 표(본부 합계 행 포함)

### 1.2 동작 규칙
- 기준일은 실시간 오늘 날짜 기준 표시.
- 차트 기간 필터: 올해, 작년, 전체기간 + 연도 선택.
- WIN/LOSS는 올해 누적 기준 카드로 표시.

---

## 2. 업무수행현황 (`/projects/operations`)

### 2.1 화면 구성
- 상단 필터: 사업유형, 상태, 영업대표, PM, 기간, 검색어
- 액션: 조회, 초기화, 신규 프로젝트 등록
- KPI 카드: 전체 프로젝트, 제안 단계, 수행 단계, 종료
- 사업 목록 테이블
- 테이블 우측 상단: 검색결과 수, 필터 조건, 엑셀 다운로드

### 2.2 기간 필터
- 프리셋: 전체, 최근 3개월, 이번달, 지난달, 올해, 직접 선택
- 직접 선택: 시작일/종료일 범위를 달력에서 선택
- 기준 필드: 제안 제출일(`submission.datetime`)

### 2.3 검색/필터 동작
- 드롭다운/입력값은 폼 상태에 유지되고 `조회` 클릭 시 적용.
- 검색어 엔터 입력 시 조회 실행.
- 상단 KPI 카드 선택 필터와 상단 필터를 동시 적용.

### 2.4 목록 테이블 컬럼
- 상태, 사업명, 고객사, 영업대표, 제안PM, 발표PM, 수행PM, 제안/수행팀, 사업 종료일, 프로젝트 상세

### 2.5 다운로드
- 버튼명: `엑셀 다운로드`
- 파일명: `업무수행현황_YYMMDD_hhmm.csv`
- 대상: 현재 필터 결과 전체(페이지네이션 현재 페이지 아님)
- 내보내기 컬럼 체계: 프로젝트 관리 테이블 기준 그룹 헤더/상세 헤더 포함

---

## 3. 프로젝트 관리 (`/projects/codes`)

### 3.1 화면 구성
- 상단 필터
  - 1행: 영업대표, 제안PM, 사용여부
  - 2행: 기간, 검색어, 조회, 초기화, 신규 프로젝트 등록
- 상태 카드(8개)
- 프로젝트 목록 테이블(2단 헤더)
- 하단 페이지네이션

### 3.2 기간 필터
- 프리셋: 전체, 최근 3개월, 이번달, 지난달, 올해, 직접 선택
- 직접 선택: 시작일/종료일 범위 선택
- 기준 필드: 사업 시작일(`fromDate`)

### 3.3 목록 정렬/필터
- 기본 정렬: 코드 내림차순
- 카드 필터 + 상단 필터 동시 적용
- 필터 조건은 테이블 우측 상단에 텍스트로 표시

### 3.4 목록 테이블 헤더/컬럼
- 1행 그룹
  - 프로젝트 코드/사업명
  - 프로젝트 기본정보
  - 인력 정보
  - 프로젝트 일정
  - 기타
- 2행 컬럼
  - 코드, 사업명
  - 고객사, 사업유형, 확도, 사업금액, 공고번호, 공고일, 상태
  - 영업부서, 영업대표, 제안PM, 발표PM, 수행PM, 제안/수행팀
  - 사업 시작일, 사업 종료일, 제안서 제출일, 제출 형식, 제출 유의사항, 제안 발표일, 발표 형식, 발표 유의사항
  - 최근활동일, 사용여부

### 3.5 다운로드
- 버튼명: `엑셀 다운로드`
- 파일명: `프로젝트관리_YYMMDD_hhmm.csv`
- 대상: 현재 필터 결과 전체
- 출력 포맷: 그룹 헤더(1행) + 컬럼 헤더(2행) + 데이터

### 3.6 라우팅 정합
- `/projects/codes` 라우트에서 `searchParams`를 파싱해 초기값 전달
- 클라이언트 페이지는 `useSearchParams` 없이 `initialCreate`, `initialEditCode` props 사용

## 4. 프로젝트 상세 (`/projects/{projectId}`)

### 4.1 화면 구성
- 기본정보/일정/인력 정보/기타
- 투입 인력/참여 정보 테이블
- 진행이력 테이블

### 4.2 적용 규칙
- `인력 배치 이력 보기` 버튼 제거됨
- 진행이력의 작성자/변경자 표시는 진행이력 페이지와 동일 규칙 적용

---

## 5. 진행이력 (`/projects/logs`)

### 5.1 화면 구성
- 상단 필터: 프로젝트, 이력 유형, 작성자/변경자, 기간, 검색어
- KPI 카드: 전체 이력, 최근 7일 등록, 최근 7일 상태 변경, 활성 프로젝트
- 진행이력 목록 테이블
- 프로젝트별 이력 건수 테이블

### 5.2 기간 필터
- 프리셋: 전체, 최근 3개월, 이번달, 지난달, 올해, 직접 선택
- 직접 선택: 시작일/종료일 범위 선택
- 기준 필드: 이력 일시(`datetime/logged_at`)

### 5.3 작성자/변경자 출력 규칙
- `author_name == updated_by_name`: 작성자 1개만 표시
- 다름: `author_name/updated_by_name` 형식 표시(셀 줄바꿈 허용)

### 5.4 검색/필터 동작
- 검색어 엔터 입력 시 조회 실행
- 상단 필터 적용 시에도 테이블 우측 상단에 필터 조건 텍스트 표시

---

## 6. 공통 구현 규칙

### 6.1 상태 전환
- 제안중 → 발표완료 / DROP
- 발표완료 → WIN / LOSS
- WIN → 수행중
- 수행중 → 완료
- 업무지원 → 완료

### 6.2 제출 형식 SSOT
- 제출 형식 옵션: `온라인`, `오프라인`, `이메일`
- SSOT 상수: `apps/web/design/constants/projectFormOptions.ts`의 `SUBMISSION_FORMAT_OPTIONS`

### 6.3 프로젝트 코드 생성
- 형식: `P{YYYY}{3자리 시퀀스}` (예: `P2026001`)

### 6.4 기간 선택 컴포넌트
- 공통 기간 선택 UI는 `CommonPeriodPicker`를 사용
- `직접 선택` 모드에서 시작일/종료일 범위 선택 지원

---

## 7. 주요 API/화면 데이터 소스

- 홈/업무수행현황/프로젝트 관리/진행이력: `GET /api/p1-screens/{screen}`
- 프로젝트 생성: `POST /api/projects`
- 프로젝트 수정: `PATCH /api/projects/{id}`
- 프로젝트코드 생성: `POST /api/project-codes`
- 프로젝트코드 수정: `PATCH /api/project-codes/{id}`

---

## 8. 검증 기준 (현행)

- 페이지별 필터(`조회/초기화/엔터 조회`)가 일관되게 동작한다.
- 직접 선택 기간이 각 화면 기준 필드에 맞게 필터링된다.
- 다운로드는 항상 필터 결과 전체를 내보내며 파일명 규칙을 따른다.
- 프로젝트 관리 페이지 빌드 시 `useSearchParams` suspense 오류가 재발하지 않는다.
