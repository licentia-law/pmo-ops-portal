# DTL P1 — 핵심 운영 원장

- 작성일시: 2026-05-08 07:02
- 범위: 업무수행현황, 프로젝트코드, 프로젝트상세, 진행이력
- 목표: 프로젝트 운영 원장과 상세/이력 흐름을 먼저 구현해 시스템의 핵심 업무 흐름을 확보한다.

---

## 1. 업무수행현황

### 1.1 목표
기존 `업무수행현황` 시트의 원장 역할을 브라우저 목록형 화면으로 재현한다. 사업 기본 정보, 일정, PM/영업대표, 진행 메모, 상태를 한 화면에서 조회하고 등록/수정의 진입점으로 사용한다.

### 1.2 선행 데이터
- projects
- project_codes
- organizations
- employees
- 상태값 enum

### 1.3 화면 구성
- 상단 필터: 본부, 팀, 상태, 사업유형, 제안PM, 영업대표, 기간기준, 기간, 검색어
- 액션: 신규 프로젝트 등록, 엑셀 다운로드
- KPI: 전체 프로젝트, 제안중, 수행중, 종료
- 메인 테이블
- 우측 선택 프로젝트 상세 패널(선택 구현)

### 1.4 테이블 컬럼
- 프로젝트코드
- 사업명
- 사업유형
- 상태
- 사업금액
- 제안PM
- 영업대표
- 총괄부서
- 시작일
- 완료일
- 메모
- 수정일시
- 수정자
- 작업

### 1.5 입력값
- 필수: 사업명, 프로젝트코드(자동), 사업금액, 사업유형, 시작기준일, 지원완료일, 주관부서, 영업대표, 제안PM, 상태값
- 선택: 공고번호, 공고일, 세부 비고, 진행 메모, 발표/제출 시간, 방식, 비고

### 1.6 출력/계산
- 상태 badge
- 사업금액 포맷
- 기간기준별 조회
- 검색 결과 수

### 1.7 동작 규칙
- 행 클릭 시 프로젝트상세 또는 우측 패널
- 신규 등록은 별도 등록 화면 또는 상세 페이지 신규 모드
- 상태 변경은 상세 페이지 중심으로 수행
- 목록에서는 최소 수정/상세 진입만 제공

### 1.8 API 초안
- `GET /api/projects`
- `POST /api/projects`
- `GET /api/projects/{id}`
- `PATCH /api/projects/{id}`
- `POST /api/reports/project-ledger/export`

### 1.9 예외 처리
- 코드 중복 방지
- 잘못된 상태 전환 방지
- 필수 일정 누락 시 저장 불가

### 1.10 DoD
- 엑셀 원장의 핵심 컬럼이 누락 없이 조회된다.
- 목록에서 상세 진입이 가능하다.
- 필터가 동작한다.
- 신규 등록이 가능하다.

---

## 2. 프로젝트코드

### 2.1 목표
기존 `프로젝트코드` 시트 및 KPI 프로젝트 마스터 시트 성격을 브라우저 프로젝트 마스터 화면으로 재현한다.

### 2.2 화면 구성
- 상단 필터: 상태, 영업대표, 사용여부, 검색어
- 액션: 조회, 초기화, 신규 프로젝트 등록
- 상태별 요약 카드
- 메인 테이블

### 2.3 테이블 컬럼
- 프로젝트코드
- 프로젝트명
- 상태
- 확도/구분(필요 시)
- 영업부서
- 영업대표
- 제안PM
- 생성일
- 사용여부
- 비고
- 작업

### 2.4 동작 규칙
- 프로젝트코드는 자동 부여가 기본
- 필요 시 관리자 수기 수정 허용
- 사용여부 비활성 가능

### 2.5 API 초안
- `GET /api/project-codes`
- `POST /api/project-codes`
- `PATCH /api/project-codes/{id}`

### 2.6 DoD
- 프로젝트 코드 목록과 상태가 조회된다.
- 신규 등록/수정이 가능하다.
- 사용여부 관리가 가능하다.

---

## 3. 프로젝트상세

### 3.1 목표
한 프로젝트의 기본정보, 일정정보, 핵심지표, 투입인력, 최근 진행사항을 한 화면에서 확인/수정한다.

### 3.2 화면 구성
- 상단: 프로젝트명, 상태 badge, 사업유형, 사업금액, 상태 전환 드롭다운, 수정/목록 버튼
- 카드 1: 기본정보
- 카드 2: 일정정보
- 카드 3: 핵심지표
- 섹션 4: 투입 인력 / 참여 정보
- 섹션 5: 최근 진행사항

### 3.3 상세 항목
- 프로젝트코드
- 사업명
- 사업유형
- 상태
- 제안PM
- 영업대표
- 주관부서
- 제안작성일
- 제출일/시간/방식/비고
- 발표일/시간/방식/비고
- 수행시작일
- 종료예정일

### 3.4 투입 인력 표 컬럼
- 이름
- 역할
- 소속팀
- 배정유형
- 투입상태
- 상주여부
- 시작일
- 종료일
- 비고

### 3.5 진행사항 표 컬럼
- 일시
- 내용
- 작성자
- 상태

### 3.6 동작 규칙
- 상태 전환은 공통 상태 전환 규칙 사용
- 발표완료 이후 해당 PM/제안팀만 상세 수정 가능
- 인력 배치 이력 보기 버튼 제공

### 3.7 API 초안
- `GET /api/projects/{id}`
- `PATCH /api/projects/{id}`
- `GET /api/projects/{id}/members`
- `GET /api/projects/{id}/logs`

### 3.8 DoD
- 프로젝트 요약/일정/인력/진행이력 조회 가능
- 상태 전환 동작 가능
- 수정 가능 권한 제어 가능

---

## 4. 진행이력

### 4.1 목표
프로젝트별 진행 로그와 상태 변경 이력을 기록/조회한다.

### 4.2 화면 구성
- 상단 필터: 프로젝트, 이력유형, 작성자, 기간, 검색어
- KPI: 전체 이력, 최근 7일 등록, 최근 7일 상태 변경
- 메인 테이블: 진행 이력 목록
- 우측 패널: 최근 상태 변경, 프로젝트별 이력 건수, 선택 이력 상세

### 4.3 테이블 컬럼
- 일시
- 프로젝트코드
- 프로젝트명
- 이력 유형
- 요약 내용
- 작성자
- 상세 보기

### 4.4 이력 유형 예시
- 상태 변경
- 투입 인원 변경
- 발표 일정 등록
- 진행 메모
- 업무지원 등록
- 일정 변경

### 4.5 동작 규칙
- 상세 페이지와 연결
- 선택 이력 상세 표시
- 상태 변경 이력은 before/after 상태를 별도 표시 가능

### 4.6 API 초안
- `GET /api/project-logs`
- `POST /api/project-logs`
- `GET /api/project-logs/{id}`

### 4.7 DoD
- 프로젝트별 이력 조회 가능
- 필터/기간 검색 가능
- 선택 이력 상세 확인 가능

1. 프로젝트 도메인 타입 정의

3. 프로젝트코드 마스터 페이지 구현
4. 프로젝트상세 페이지와 진행이력 페이지 연결
5. 공통 상태 badge와 상태 전환 util 작성
