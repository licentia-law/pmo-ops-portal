# DTL P2 — 인력 운영 핵심

- 작성일시: 2026-05-08 07:02
- 범위: 인력재직현황, 인력배치/투입현황, 인원별 투입(현재)
- 목표: 재직 마스터, 인력×프로젝트 팩트, 기준일 스냅샷을 구현해 MM 집계의 기반을 만든다.

---

## 1. 인력재직현황

### 1.1 목표
KPI 파일의 `인력재직현황` 시트를 브라우저형 인력 마스터 및 연간 재직 MM 관리 화면으로 재현한다.

### 1.2 화면 구성
- 상단 필터: 본부, 팀, 재직여부, 직위, 기준일, 기준연도, 검색어
- 액션: 조회, 초기화, 엑셀 다운로드, 신규 인력 등록
- KPI: 전체 인력, 재직, 휴직, 전배, 퇴직, 연간 재직 MM
- 인력 목록 테이블
- 월별 재직 MM 요약
- 상태별 인원 요약

### 1.3 테이블 컬럼
- No
- 본부
- 팀
- 성명
- 직위
- 재직여부
- 입사일자
- 시작일
- 만료일
- 연간 재직 MM
- 비고
- 작업

### 1.4 계산
- 월별 재직 MM 합산
- 재직/휴직/전배/퇴직 인원 집계

### 1.5 API 초안
- `GET /api/employees`
- `POST /api/employees`
- `PATCH /api/employees/{id}`
- `GET /api/employees/{id}/employment-history`

### 1.6 DoD
- 인력 마스터 조회/등록/수정 가능
- 재직 상태 변경 가능
- 연간 재직 MM이 조회 가능

---

## 2. 인력배치/투입현황

### 2.1 목표
`인력_투입현황MM`의 사람×프로젝트×기간 단위 fact를 브라우저에서 조회/수정 가능하게 한다.

### 2.2 화면 구성
- 상단 필터: 기준월, 구분, 본부, 팀, 투입여부, 상주여부, 검색어
- KPI: 총 배정 인원, 수행 MM, 제안 MM, 지원 MM, 미투입 MM
- 상단 배정 현황 타임라인(주차 기준)
- 우측 배정 요약 패널
- 하단 배정 상세 목록

### 2.3 상세 목록 컬럼
- No
- 성명
- 프로젝트코드
- 프로젝트명
- 영업대표
- 구분
- 투입여부
- 상주여부
- 시작일
- 만료일
- 이번달 MM
- 확도
- 역할
- 비고
- 관리

### 2.4 동작 규칙
- 신규 배정 등록 가능
- 중복 배정 경고
- 과부하/외부인원/중복배정 요약 표시
- 타임라인은 조회 중심, 하단 표는 수정 진입점

### 2.5 API 초안
- `GET /api/staff-allocations`
- `POST /api/staff-allocations`
- `PATCH /api/staff-allocations/{id}`
- `GET /api/staff-allocations/workload`

### 2.6 DoD
- 기준월 기준 배정 현황 조회 가능
- 월별 MM 조회 가능
- 신규/수정 가능
- 중복 배정 탐지가 가능

---

## 3. 인원별 투입(현재)

### 3.1 목표
기준일 현재 사람별 대표 상태와 현재 프로젝트를 보여주는 스냅샷 화면을 구현한다.

### 3.2 화면 구성
- 상단 필터: 조회 기준일, 본부, 팀, 대표 상태, 검색어
- KPI: 전체 인원, 수행중, 제안중, 지원, 대기, 휴직, 전배, 퇴직, 가동률(선택)
- 현재 인원 스냅샷 테이블
- 월별 MM(상세)
- 상태별 인원 요약
- 차기 투입 예정

### 3.3 테이블 컬럼
- No
- 성명
- 본부/팀
- 대표 상태
- 현재 프로젝트
- 현재 구분
- 역할
- 참여기간
- 상주여부
- 차기 예정 프로젝트
- 시작예정일
- 작업

### 3.4 동작 규칙
- 대표 상태는 기준일 현재 우선 상태 기준
- 차기 예정 프로젝트는 다음 시작예정일 기준
- 상태별 요약과 현재 스냅샷의 기준일이 일치해야 함

### 3.5 API 초안
- `GET /api/staff-current-snapshot`
- `GET /api/staff-current-snapshot/summary`

### 3.6 DoD
- 기준일 현재 상태가 조회된다.
- 현재 프로젝트/차기 예정 프로젝트가 구분된다.
- 상태별 요약이 표시된다.
