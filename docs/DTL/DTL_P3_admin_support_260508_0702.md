# DTL P3 — 운영 보조 및 관리자 기반

- 작성일시: 2026-05-08 07:02
- 범위: 대기현황, 사용자/권한 관리, 기준정보 관리, 공휴일 관리
- 목표: 운영 보조 화면과 관리자 마스터 화면을 구현해 시스템 운영 기반을 만든다.

---

## 1. 대기현황

### 1.1 목표
전주/금주/차주 기준의 인력 상태 운영판을 제공한다. 대기 인원 중심으로 보되 휴직/전배/퇴직도 별도 구분한다.

### 1.2 화면 구성
- 필터: 기준일, 본부, 팀, 상태, 검색어
- KPI: 전체 인원, 전주 대기 인원, 금주 대기 인원, 차주 투입 예정
- 중앙 3주 운영판 매트릭스
- 우측 비배정 인원 요약
- 우측 차주 투입 예정

### 1.3 매트릭스 컬럼 구조
- No
- 성명
- 역할
- 전주(대기/휴직/전배/퇴직)
- 금주(대기/휴직/전배/퇴직)
- 차주(대기/휴직/전배/퇴직/투입예정)

### 1.4 동작 규칙
- 대기 인원은 `대기` 상태만 집계
- 휴직/전배/퇴직은 별도 구분
- 프로젝트명/역할/계약기간/현재정보/차기예정 정보 hover 또는 상세로 확인 가능

### 1.5 API 초안
- `GET /api/bench-status`
- `GET /api/bench-status/summary`

---

## 2. 사용자/권한 관리

### 2.1 목표
로컬 계정 기반 사용자 생성, 권한 부여, 데이터 범위 관리 기능을 제공한다.

### 2.2 화면 구성
- 필터: 권한, 소속 본부, 소속 팀, 상태, 검색어
- KPI: 전체 사용자, 활성 사용자, 관리자/본부장, 비활성 사용자
- 사용자 목록
- 우측 사용자 상세 패널
- 권한 레벨 안내

### 2.3 목록 컬럼
- 이름
- 아이디
- 소속본부/팀
- 조직역할
- 권한
- 데이터 범위
- 상태
- 최근 접속
- 작업

### 2.4 상세 패널 항목
- 이름
- 이메일/아이디
- 소속 본부/팀
- 조직역할
- 권한
- 데이터 범위
- 최근 접속
- 비고
- 관리자만 권한 변경 및 저장 가능 안내

### 2.5 권한 레벨 안내
- 조회 전용: 조회/다운로드만 가능
- 일반 수정 가능: 일반 데이터 수정 가능
- 프로젝트 담당 수정 가능: 발표완료 이후 본인 담당 프로젝트 상세/투입기간 수정 가능
- 관리자: 사용자/권한/기준정보/시스템 설정 가능

### 2.6 API 초안
- `GET /api/users`
- `POST /api/users`
- `PATCH /api/users/{id}`
- `POST /api/users/{id}/password-reset`

## 4. 공휴일 관리

### 4.1 목표
달력 시트의 공휴일 마스터를 시스템에서 관리하고 MM 계산에 반영한다.

### 4.2 화면 구성
- 필터: 연도, 월, 구분, 검색어
- 안내바: 올해 공휴일 수 / 선택월 공휴일 수 / 월간 MM 계산 기준 안내
- 월간 캘린더
- 공휴일 목록
- 공휴일 상세 폼

### 4.3 목록 컬럼
- 일자
- 명칭
- 구분
- 계산반영
- 작업

### 4.4 상세 폼
- 일자
- 명칭
- 구분
- 계산 반영
- 비고

### 4.5 동작 규칙
- 등록/수정된 공휴일은 저장 후 월별 MM 계산에 반영
- 대체공휴일 구분 가능

### 4.6 API 초안
- `GET /api/holidays`
- `POST /api/holidays`
- `PATCH /api/holidays/{id}`
- `DELETE /api/holidays/{id}`
