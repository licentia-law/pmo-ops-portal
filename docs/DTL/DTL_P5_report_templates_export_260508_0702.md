## 1. 제안PRJ

### 1.1 목표
`Ⅱ.4.제안PRJ` 시트를 브라우저 보고/출력 템플릿으로 재현한다.

### 1.2 화면 구성
- 필터: 기준일(공통), 사업유형, 영업대표, 검색어
- 상단 요약: 총 제안 프로젝트, 본공고 예정, 입찰 예정, PT 예정
- 메인 표: 제안 프로젝트 보고
- 액션: 행 추가, 편집, 저장, 엑셀 다운로드

### 1.3 표 컬럼
- 유형
- 건수
- 프로젝트명
- 본공고 일정
- 입찰 일정
- PT 일정
- PM
- 제안단계
- 비고

### 1.4 동작 규칙
- 보고 화면이므로 최소한의 수기 보완 허용
- 제안단계는 보고용 단계이며 기존 진행현황과 맵핑 가능

### 1.5 API 초안
- `GET /api/reports/proposal-projects`
- `PATCH /api/reports/proposal-projects`
- `POST /api/reports/proposal-projects/export`

## 3. 보고서 다운로드

### 3.1 목표
기존 보고 시트들을 엑셀(.xlsx)로 생성/다운로드하는 허브 화면을 제공한다.

### 3.2 출력 대상
- 업무수행현황
- 프로젝트코드
- 인력재직현황
- 인력_투입현황MM
- 인원별_투입(현재)
- 대기현황
- 월별투입현황
- 주간현황
- 월별가동현황
- 대기/제안인원
- 제안PRJ
- 이행PRJ

### 3.3 화면 구성
- 1단계: 기준 선택
  - 기준일(주간/스냅샷 보고용)
  - 기준년월(월간 보고용)
  - 출력 형식(.xlsx)
  - 기준 적용 방식(자동/보고서별 기준)
- 2단계: 보고서 선택
  - 기본 현황 / 인력 현황 / KPI·보고 그룹 체크박스
- 3단계: 다운로드
  - 선택 요약
  - 최근 다운로드
  - 선택 보고서 다운로드 버튼

### 3.4 동작 규칙
- 주간/스냅샷 보고는 기준일 사용
- 월간 보고는 기준년월 사용
- 보고서별 기준 자동 적용 허용

### 3.5 API 초안
- `POST /api/reports/export-batch`
- `GET /api/reports/download-history`

### 3.6 DoD
- 선택한 보고서가 엑셀로 다운로드된다.
- 기준일/기준년월 규칙이 적용된다.
- 최근 다운로드 이력 조회 가능
