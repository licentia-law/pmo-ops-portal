# DTL P6 — 월마감/스냅샷

- 작성일시: 2026-05-08 07:02
- 범위: 월마감/스냅샷
- 목표: 특정 월 기준 데이터를 확정하고 재현 가능한 형태로 보관하는 관리자용 구조를 만든다.
- 비고: PRD상 MVP 포함 여부는 운영상 고려이지만, 최소한 구조는 반드시 반영한다.

---

## 1. 목표
- 기준월의 인력/프로젝트/MM 데이터를 확정한다.
- 이후 보고서 재현 시 확정본과 재계산본을 구분할 수 있게 한다.
- 공휴일 반영 여부, 실행자, 실행일시를 기록한다.

---

## 2. 화면 구성
- 상단 단계 표시: 월 선택 → 실행/확정 → 이력/재확인
- 필터: 기준연도, 기준월, 본부, 상태
- 액션: 조회, 초기화, 월마감 실행, 스냅샷 생성
- KPI: 기준월 상태, 최근 마감일시, 스냅샷 건수, 권한/안내
- 좌측: 월마감 이력
- 우측: 선택 월 상세 + 실행 전 점검 + 최근 스냅샷

---

## 3. 월마감 이력 테이블
컬럼:
- 기준월
- 상태
- 실행일시
- 실행자
- 스냅샷 여부
- 확정 인원수
- 확정 프로젝트수
- 확정 MM
- 대상 본부
- 작업

상태 예시:
- 마감완료
- 검토중
- 재실행필요

---

## 4. 선택 월 상세
표시 항목:
- 기준월
- 상태
- 실행 책임자
- 확정 인원수
- 확정 프로젝트수
- 확정 MM
- 공휴일 반영 여부
- 스냅샷 보관 여부

실행 전 점검:
- 인력 데이터 확인
- 프로젝트 상태 확인
- 공휴일 반영 확인
- MM 재계산 완료

버튼:
- 마감 확정
- 재실행

---

## 5. 권한 규칙
- 관리자 및 본부장만 실행 가능
- 일반 사용자는 조회만 가능
- 이미 마감완료된 월은 재실행 시 경고 필요

---

## 6. API 초안
- `GET /api/monthly-closing`
- `POST /api/monthly-closing/run`
- `POST /api/monthly-closing/{id}/snapshot`
- `POST /api/monthly-closing/{id}/rerun`

---

## 7. 예외 처리
- 기준월 미선택 시 실행 불가
- 공휴일 데이터 없음 경고
- 필수 팩트 데이터 부족 시 실행 중단
- 이미 확정된 월 재실행 시 확인 모달 필요

---

## 8. DoD
- 월별 확정 이력이 조회된다.
- 선택 월 상세와 스냅샷 이력이 조회된다.
- 관리자/본부장만 실행할 수 있다.
- 재현 가능한 월 데이터 기준이 남는다.

---

## 9. 로컬 Codex 작업 지시
1. 월마감은 초기에는 mock/placeholder 상태로 구현
2. 실행 버튼은 관리자 권한 guard 적용
3. 월별 snapshot entity와 history table 설계부터 선행
