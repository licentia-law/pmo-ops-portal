## 1. 공통 목표
본 시스템은 기존 엑셀 2종을 브라우저형 운영 시스템으로 재현하는 MVP이다. 공통 DTL의 목적은 다음과 같다.

1. 페이지별 구조가 달라도 동일한 제품 경험을 유지한다.
2. 상태값, 사업유형, 권한, 데이터 범위, 날짜 입력 방식, 다운로드 정책을 전 페이지에서 일관되게 사용한다.
3. 공통 컴포넌트를 우선 재사용하고, 페이지별 특수 컴포넌트는 최소화한다.
4. 계산의 기준이 되는 원장/마스터/팩트 데이터를 분리한다.
5. 로컬 Codex가 공통 레이아웃과 공통 UI/상태관리부터 구현할 수 있도록 기준을 제공한다.

## 4. 공통 입력 정책

### 4.1 날짜 입력 정책
기본은 날짜 입력, 필요 시 시간 포함으로 구현한다.
- 기본 필드: 날짜 선택
- 보조 옵션: 시간 포함 체크박스
- 체크 시 시간 입력칸 노출
- 적용 대표 필드: 제출일, 발표일, 진행일시, 마감 실행일시

### 4.2 검색 입력 정책
- 검색어는 최소 1개 공통 placeholder 규칙을 적용한다.
- 목록형 페이지는 주로 텍스트 통합 검색 1개를 둔다.
- 검색 대상은 페이지별 DTL에서 명시한다.

### 4.3 선택값 입력 정책
- 상태값, 사업유형, 권한, 데이터 범위는 자유입력이 아니라 코드값 dropdown으로 제한한다.
- 공휴일 구분, 계산 반영 여부, 사용여부도 코드값으로 제한한다.

### 4.4 저장 정책
- 등록/수정은 명시적 저장 버튼을 기준으로 한다.
- 자동 저장은 MVP 범위에서 사용하지 않는다.
- 보고 템플릿(제안PRJ, 이행PRJ)은 최소 수정/저장 구조를 허용한다.

## 5. 공통 계산 정책

### 5.1 가동률 / 가득률
- 주간현황: 사람 수 기준 공식 사용
- 월별가동현황: 월별 MM 기반 집계 사용
- 구체 식은 각 KPI 페이지 DTL에서 명시한다.

### 5.2 MM 계산
- 월별 MM은 주말/공휴일 제외 기준 사용
- 연차 제외 기능은 MVP 미구현
- 재직 MM과 투입 MM은 분리 계산한다.
- 공휴일 마스터는 공휴일 관리/달력 기준을 사용한다.

### 5.3 스냅샷 기준
- 인원별 투입(현재): 기준일 현재 대표 상태와 현재 프로젝트 기준
- 대기현황: 전주 / 금주 / 차주 비교 기준
- 대기/제안인원: 기준일 기준 설명형 리스트 보고

### 7.2 테이블 공통 규칙

- 컬럼 정렬은 MVP에서 필요한 컬럼만 허용

- 우측 상세패널은 관리자/이력/선택형 보고 화면에서 사용
- 목록 + 상세 동시 노출이 필요한 경우만 사용
- 상세패널이 없어도 되는 페이지는 별도 상세 페이지 사용

## 8. 공통 API 규칙

### 8.1 네이밍
- 목록 조회: `GET /api/{resource}`
- 상세 조회: `GET /api/{resource}/{id}`
- 등록: `POST /api/{resource}`
- 수정: `PATCH /api/{resource}/{id}`
- 삭제/비활성: `DELETE` 또는 `PATCH /status`
- 다운로드: `POST /api/reports/{report_type}/export`

### 8.2 응답 기본 구조
```json
{
  "items": [],
  "page": 1,
  "page_size": 10,
  "total": 216,
  "summary": {}
}
```

### 8.3 필터 파라미터 공통 규칙
- `page`, `page_size`, `sort`, `q`
- 조직 필터: `hq`, `team`
- 날짜 필터: `base_date`, `base_month`, `base_week`

## 10. 공통 검증/예외 처리
- 필수값 누락 시 저장 불가
- 권한 없는 수정 시 저장 불가
- 잘못된 상태 전환 시 저장 불가
- 기준일/기준월 미선택 시 조회 불가
- 공휴일/월마감 데이터가 없을 때 경고 노출
- 다운로드 실패 시 재시도 안내

## 11. 공통 다운로드 정책
- MVP 출력 형식: `.xlsx`
- 다운로드 전 기준일 / 기준주차 / 기준월 검증
- 보고서 종류별 기준 자동 적용 허용
- 최근 다운로드 목록은 선택 기능이며, 미구현 시 단순 다운로드 이력만 표시 가능
