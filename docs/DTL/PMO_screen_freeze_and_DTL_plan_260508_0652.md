## 1. 결론

현재 상태는 **DTL 작성으로 넘어가도 되는 단계**다.

다만 바로 전 페이지를 동시 개발하기보다,

1. 공통 규칙 고정
2. 화면 Freeze 표 확정
3. P0/P1 페이지부터 DTL 작성
4. Codex로 구현 착수

순서로 가는 것이 가장 안전하다.

### 판단 요약
- 화면 구조: 충분히 고정됨
- 정보 배치: DTL 작성 가능 수준
- 공통 디자인 시스템: `_shared` 분리 기준으로 재사용 가능
- 남은 핵심 과제: 디자인보다 **용어/컬럼/계산/권한 규칙 고정**

### 지금 더 먼저 해야 할 일
- 전체 화면을 다시 디자인하는 작업: 불필요
- 픽셀 단위 UI 보정: 후순위
- DTL 없이 바로 구현: 비추천

### 지금 가장 먼저 해야 할 일
- 본 문서 기준으로 Freeze 확정
- 공통 DTL 작성
- 업무수행현황부터 페이지 DTL 작성

## 2. Freeze 판정 기준

### Freeze 상태 정의
- **A. Freeze 완료**: DTL로 바로 이동 가능
- **B. Freeze 조건부 완료**: 작은 문구/컬럼 조정만 남음
- **C. Freeze 보류**: DTL 전에 구조 재확인 필요

### 확인 항목
- 페이지 목적이 명확한가
- 상단 필터가 확정됐는가
- KPI 카드가 확정됐는가
- 메인 테이블/카드 구조가 확정됐는가
- 버튼이 확정됐는가
- 상세 영역 방식이 확정됐는가
- PRD 요구 기능이 누락되지 않았는가
- 구현 관점에서 과하지 않은가

## 4. 페이지별 상세 Freeze 체크

## 4.1 홈
### 판단
Freeze 완료.

### 확정 항목
- 상단 소개 영역 유지
- 주요 화면 바로가기 카드 유지
- 핵심 현황 카드 유지
- 최근 진행 프로젝트 + 이번 달 요약 2단 구조 유지

### DTL로 넘길 항목
- 바로가기 카드 클릭 라우팅
- 최근 진행 프로젝트 데이터 소스
- 이번 달 요약 집계 기준

## 4.2 대시보드
### 판단
Freeze 완료.

### 확정 항목
- 필터: 본부 / 팀 / 기간 / 상태
- KPI: 현재 인원 / 수행 인원 / 제안 인원 / 대기 인원 / 가동률 / 가득률
- 최근 6개월 가동률/가득률 추이
- 팀별 현황 요약
- 팀별 현황(가동률 기준)

### DTL로 넘길 항목
- KPI 집계 기준
- 추이 차트 API 형식
- 팀별 집계 기준

## 4.3 업무수행현황
### 판단
Freeze 조건부 완료.

### 현재 구조 평가
- 원장형 메인 화면 방향은 적절
- 상단 필터 + KPI + 목록 구조는 유지 추천
- 우측 상세패널 포함 여부는 구현 난도에 영향 큼

### 확정 권장 항목
- 상단 필터
  - 본부
  - 팀
  - 상태값
  - 사업유형
  - 제안PM
  - 영업대표
  - 기간 기준
  - 기간
  - 검색어
- 버튼
  - 신규 프로젝트 등록
  - 엑셀 다운로드
- KPI
  - 전체 프로젝트
  - 제안중
  - 수행중
  - 종료
- 메인 테이블 핵심 컬럼
  - 프로젝트코드
  - 사업명
  - 사업유형
  - 상태
  - 사업금액
  - 제안PM
  - 영업대표
  - 총괄부서
  - 시작일
  - 완료일
  - 메모
  - 수정일시
  - 수정자
  - 작업

### DTL 전 마지막 고정 필요
- 우측 상세패널을 유지할지, 별도 상세 페이지로만 갈지
- 상태값 enum을 최종 상태값 8종으로 고정할지
- 제안PM / 발표PM / 제안팀 표시 범위를 어디까지 둘지

### 권장
- 업무수행현황은 **목록 중심 + 상세 버튼 + 프로젝트상세 이동** 기준으로 DTL 작성
- 우측 상세패널은 1차 구현에서 옵션으로 두는 것이 안전

## 4.5 프로젝트상세

### 확정 항목

### 권장
- 프로젝트상세는 DTL에서 반드시
  - 읽기 전용 정보
  - 수정 가능 정보
  - 발표완료 이후 수정 가능 정보
  를 구분해 기술해야 함

## 4.7 대기현황
### 판단
Freeze 보류.

### 이유
- 현재 화면은 운영판으로는 좋지만 구현 난도가 높음
- 전주/금주/차주 3주 매트릭스는 데이터 모델과 셀 규칙을 먼저 확정해야 함
- 스냅샷/예정/상태변화가 한 화면에 동시에 많음

### 유지 추천 구조
- 상단 필터
- KPI 카드
- 중앙 3주 운영판
- 우측 요약 2패널

### DTL 전 선행 필요
- 셀 단위 표시 규칙
- 상태별 색상/표시 우선순위
- 대기/휴직/전배/퇴직의 포함 기준
- 차주 투입 예정 정의

### 권장
- 대기현황은 DTL은 쓰되 **P2 또는 P3**로 두는 것이 좋음

## 8. DTL 초안 구조

아래 초안은 Codex와 사람이 바로 이어서 확장할 수 있도록 만든 시작점이다.

## 8.1 공통 DTL 초안

### 목표
- 모든 페이지가 공통 레이아웃과 공통 규칙을 사용하도록 고정

### 포함 항목
- 좌측 메뉴 구조
- 상단 헤더 공통 구조
- 공통 카드 컴포넌트
- 공통 테이블 컴포넌트
- 공통 상태 배지
- 공통 필터 바
- 공통 페이지네이션
- 공통 빈 상태 / 오류 상태 / 로딩 상태

### 공통 enum
- 상태값: 제안중 / 발표완료 / WIN / LOSS / DROP / 수행중 / 업무지원 / 완료
- 사업유형: 주사업 / 부사업 / 하도 / 협력
- 권한: 조회 전용 / 일반 수정 가능 / 프로젝트 담당 수정 가능 / 관리자
