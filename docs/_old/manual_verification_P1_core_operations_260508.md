# P1 핵심 운영 기능 수동 검증 시나리오

## 실행
1. API 환경 구성 후 `apps/api`에서 `alembic upgrade head`를 실행한다.
2. 루트에서 `npm run dev:web`를 실행한다.
3. 별도 터미널에서 `npm run dev:api`를 실행한다.
4. 브라우저에서 `http://localhost:3000/projects/operations`를 연다.

## 업무수행현황
1. 사업명, 고객사, PM, 기간, 사업유형을 입력하고 `신규 등록`을 누른다.
2. 목록에 자동 코드 `PMO-0001` 형태로 프로젝트가 표시되는지 확인한다.
3. 검색어/상태/사업유형 필터 후 `조회`를 눌러 결과가 줄어드는지 확인한다.
4. `제안중` 프로젝트의 상태 전환 선택지에 `발표완료`, `DROP`만 보이는지 확인한다.
5. `제안중 -> 수행중`은 UI 선택지에 없어야 하며, API 직접 호출 시 400 응답이어야 한다.

## 프로젝트코드
1. 프로젝트명, 담당, 사업유형, 상태를 입력하고 `신규 등록`을 누른다.
2. 목록에서 `수정`을 누른 뒤 담당/상태를 변경하고 `수정 저장`을 누른다.
3. 중복 코드를 수기 입력하면 API가 409 응답을 반환하는지 확인한다.

## 프로젝트상세
1. 업무수행현황 목록에서 프로젝트명을 눌러 상세로 이동한다.
2. 사업명, 고객사, PM, 일정, 사업유형을 수정하고 포커스를 잃으면 저장되는지 확인한다.
3. 상태 전환 선택지가 현재 상태의 허용 다음 상태로만 제한되는지 확인한다.
4. 상태 변경 후 하단 진행이력에 상태 변경 로그가 추가되는지 확인한다.

## 진행이력
1. 프로젝트를 선택하고 진행 메모를 입력한 뒤 `이력 등록`을 누른다.
2. 진행이력 목록에 신규 로그가 표시되는지 확인한다.
3. `상세`를 눌러 우측 패널에 프로젝트, 상태, 작성자, 내용이 표시되는지 확인한다.

## 권한
1. `x-user-permission: read_only` 헤더로 POST/PATCH를 호출하면 403이어야 한다.
2. `x-user-permission: general_editor` 헤더는 일반 등록/수정이 가능해야 한다.
3. `x-user-permission: project_editor`, `x-user-name: {PM명}` 헤더는 발표완료 이후 본인 PM 프로젝트만 PATCH 가능해야 한다.
4. `x-user-permission: admin` 헤더 또는 기본 개발 사용자 상태에서는 전체 수정이 가능해야 한다.
