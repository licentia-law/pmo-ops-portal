### 4.3 핵심 입력/마스터/기준 시트

#### 4.3.1 `인력재직현황` — 인력 마스터 + 연간 재직 MM

- 범위: `A1:Z26`
- 핵심 테이블명: `인력_프로젝트투입현황MM246` (`B5:Z23`)
- 실질 데이터 행: **17명 + 요약 1행**

| 열 | 의미 |
| --- | --- |
| B | No |
| C | 그룹 |
| D | 직위 |
| E | 성명 |
| F | 재직여부 |
| G | 입사일자 |
| H | 비고 |
| I | 시작일 |
| J | 만료일 |
| K | 동일월 |
| L | 사내단가 |
| M | MM |
| N:Y | 1월~12월 |
| Z | 합계 |

주요 계산식:

```excel
No = ROW() - ROW(테이블 헤더)
동일월 = IF(AND(YEAR(시작일)=YEAR(만료일), MONTH(시작일)=MONTH(만료일)), 1, 0)
각 월 MM(1월~12월) =
  시작일/만료일이 해당 월과 겹치는 정도를
  NETWORKDAYS(월 시작~월 종료, 공휴일) 기준으로
  월 근무가능일 비율로 계산
합계 = SUM(1월:12월)
```

의미:

- 이 시트는 **프로젝트 투입 여부와 무관하게 해당 인력이 그 달에 재직 중인지**를 MM 단위로 계산한다.
- `달력!B3:B26`의 공휴일을 제외한 `NETWORKDAYS`를 사용한다.
- 즉, **재직 인원 모수(분모)** 를 만드는 시트다.

#### 4.4.1 열별 역할

#### 4.4.2 핵심 계산식

1) 인력 정보 역조회

```excel
직위 = INDEX(인력재직현황!D:D, MATCH(성명, 인력재직현황!E:E, 0))
부서 = INDEX(인력재직현황!C:C, MATCH(성명, 인력재직현황!E:E, 0))
재직 = INDEX(인력재직현황!F:F, MATCH(성명, 인력재직현황!E:E, 0))
```

2) 프로젝트 정보 역조회

```excel
프로젝트코드 = VLOOKUP(프로젝트명, 프로젝트코드!B:C, 2, 0)
영업부서     = VLOOKUP(프로젝트명, 프로젝트코드!B:F, 5, 0)
```

3) 월별 MM 계산

```excel
해당 월 MM =
  계약시작일/계약만료일과 해당 월이 겹치는 비율을
  NETWORKDAYS(월 시작~월 종료, 공휴일) 기준으로 계산
```

구조적으로는 다음 3가지 경우를 나눈다.

- 시작일이 해당 월 안에 있을 때
- 해당 월 전체를 덮을 때
- 만료일이 해당 월 안에 있을 때

4) 합계 / 현재까지

```excel
합계4 = SUM(1월:12월)
현재까지 = 기준일(Ⅱ.1!L3)과 계약기간이 겹치는 월 fraction
```

5) 확도(`AH`) 가중치

```excel
IF(프로젝트코드 공란 or 미투입, 공란,
  IF(win/Loss='수주' OR (투입 and 제안), 100%,
    IF(프로젝트 마스터 상태='Sold', 100%,
      IF(프로젝트 마스터 상태='Sunk', 0%,
        IF(확도='우세', 75%,
          IF(확도='경쟁', 50%,
            IF(확도='열세', 25%,
              IF(확도='확보', 100%, 0%))))))))
```

**중요 해석:**

- 이 수식은 현재 프로젝트 마스터의 실제 값(`수주/실주/제안/Drop/취소`)과 완전히 일치하지 않는다.
- 즉, `Sold/Sunk/우세/경쟁/열세/확보`는 **과거 버전 로직의 잔재**로 보인다.
- 현재 값이 작동하는 이유는 `win/Loss='수주'` 또는 `투입 and 제안 => 100%` 같은 앞단 조건이 먼저 먹기 때문이다.

4) 사람별 월별 제안/수행 MM(`O:BA`)

- 범위: `A1:AP21`
- 목적: 기준일 기준 **전주 / 금주 / 차주** 사람별 상태를 날짜 단위로 보여주는 운영 스냅샷

핵심 계산:

```excel
기준일 = Ⅱ.1.주간현황!L3
전주 월~금 = 기준일 - WEEKDAY(기준일) - 5 ~ -1
금주 월~금 = 기준일 - WEEKDAY(기준일) + 1 ~ +5
```

- 각 날짜별 활성 assignment No를 `SUMPRODUCT`로 찾고,

### 4.6 월간 집계 엔진: `월별투입현황`

- 범위: `A1:CC18`
- 역할: KPI의 핵심 숫자(수행/제안/지원/미투입, 가동율, 가득율)를 **월별 블록 구조**로 계산

| 열 | 의미 |
| --- | --- |
| B | 구분 |
| C | 가동인원(월평균) |
| D:I | 1월 블록 = 수행/제안/지원/미투입/가동율/가득율 |
| J:O | 2월 동일 구조 |
| P:U | 3월 동일 구조 |
| V:AA | 4월 동일 구조 |
| AB:AG | 5월 동일 구조 |
| AH:AM | 6월 동일 구조 |
| AN:AS | 7월 동일 구조 |
| AT:AY | 8월 동일 구조 |
| AZ:BE | 9월 동일 구조 |
| BF:BK | 10월 동일 구조 |
| BL:BQ | 11월 동일 구조 |
| BR:BW | 12월 동일 구조 |
| BX:CC | Sold 기준 1~12월 블록 |

핵심 수식 패턴(예: 1월 블록, `D:I`):

```excel
가동인원(월평균) = SUMPRODUCT(인력재직현황[1월:12월]) / 12
수행 MM = SUMPRODUCT((부서=구분)*(투입/미투입<>'미투입')*(제안/수행='수행')*해당월MM*확도)
제안 MM = SUMPRODUCT((부서=구분)*(투입/미투입<>'미투입')*(제안/수행='제안')*해당월MM*확도)
지원 MM = SUMPRODUCT((부서=구분)*(투입/미투입='미투입')*(제안/수행='지원')*해당월MM)
미투입 MM = SUMPRODUCT(재직MM) - 수행 - 제안 - 지원
가동율 = (수행 + 제안) / (미투입 + 수행 + 제안 + 지원)
가득율 = 수행 / (미투입 + 수행 + 제안 + 지원)
```

**중요 해석:**

- 여기서 `지원`은 현재 수식상 `투입/미투입='미투입'` 조건과 결합돼 있어, KPI 철학상 **비가동 쪽**에 가깝게 취급된다.
- 즉 이 파일에서는 `지원`을 운영상 '업무는 있으나 가동률 분자에는 넣지 않는 항목'으로 처리하려는 흔적이 분명하다.

- 본문은 `유형/건수/프로젝트/일정(본공고/입찰/PT)/PM/진행현황` 구조의 **수기 보고 템플릿**이다.
- 현재 데이터는 비어 있다.

#### 4.7.5 `Ⅱ.5.이행PRJ`

- 범위: `A1:G42`
- 수식은 기준일 1개 + `SI/SM` 합계용 `SUM` 3개뿐이다.
- 구조:
  - 구분
  - 건수(전월/증감/현재)
  - 신규 프로젝트
  - 종료(예정) 프로젝트
- 현재 값은 대부분 0 또는 공란으로 남아 있어 **수기 보고용 골격**에 가깝다.
